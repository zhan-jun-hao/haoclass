```java
.set(Course::getBuyCount, this.getById(id).getBuyCount() + 1)
```

有问题，因为这条语句是，先查再更新，这样就存在丢失更新问题，假如同时有两个用户同时查询再更新，这样就只会更新一条，丢失更新

新写法：注意需要mp3.5.6版本

```java
LambdaUpdateWrapper<Course> wrapper = Wrappers.lambdaUpdate(Course.class)
        .eq(Course::getId, id)
        .eq(Course::getDeleted, 0)
        .setIncrBy(Course::getBuyCount, 1)
        .set(Course::getUpdateTime, LocalDateTime.now())
        .set(Course::getUpdatedUser, SecurityUserHolder.getUserId());
```



# 问题1：什么要改成数据库原子性

```java
.setSql("`buyCount` = `buyCount` + 1")
```

这样MySQL自带的行锁，会对这一行加行锁，保证更新

# 问题2：为什么不用RLock

因为我们只更新一个小小的字段，RLock成本要高一些，所以RLock更适合锁住较复杂的业务步骤，比如跨服务
